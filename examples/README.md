# Example Standards

Examples in this repository use generic enterprise domains such as orders, payments, policies, claims, reports, and background jobs. They are intentionally embedded in the relevant documentation so code and engineering context cannot drift apart.

An example should:

- use meaningful names and Microsoft C# conventions;
- show cancellation, validation, ownership, or failure behavior where relevant;
- avoid fake infrastructure and unexplained abstraction;
- remain small enough to review while compiling logically in its stated application context.

Start with [C# for Maintainable Backend Services](../docs/CSharp/BackendCSharp.md) or [Production REST API Design](../docs/AspNetCore/RestApiDesign.md).
